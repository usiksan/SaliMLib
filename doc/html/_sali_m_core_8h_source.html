2  SaliMLib - cooperative Minimal Multitasking Library for 32-bit single-core Microcontrollers
3 
4  Author
5  Sibilev A.S.
6 
7  www.salilab.ru
8  www.salilab.com
9  Description
10  SaliMLib - is cooperative Minimal Multitasking Library for 32-bit single-core Microcontrollers
11 
12  SaliMLib is a library for use as a multitasking framework for embedded systems based on 32-bit
13  single-core microcontrollers. This library uses cooperative multitasking. This means that each
14  task is not allocated a pre-defined time quantum, but the task itself determines the appropriate
15  time to transfer control to another task. This approach greatly simplifies task synchronization
16  and eliminates the need to use critical sections.
17 
18  The main disadvantages of cooperative multitasking compared to preemprtive multitasking are the
19  uncertainty of the response time to events, which is why such systems are not used in real-time
20  systems. However, this limitation can be easily circumvented by using hardware interrupts, thus
21  obtaining a complete analog of "hard time systems", and the actual "real time" can be achieved
22  by configuring the microcontroller peripherals to automatically work in parallel with the
23  microcontroller core.
24 
25  This library is an excellent alternative or addition to the traditional "state machines"for
26  embedded systems. With this multitasking library, the implementation of many algorithms is
27  radically simplified.
28 
29  Features
30  - minimalism. This library consumes less resources than other RTOS
31  - multitasking. The library provides cooperative multitasking
32  - a two-level priority. In the library, all tasks are divided into critical and other tasks.
33  Tasks with a critical priority level are managed first
34  - a model without priorities can be used
35  - no task scheduler. Based on the principle of cooperation all tasks of the same priority are
36  performed sequentially
37  - computer language. It uses C++ 11 for the interface and platform-independent part, as well
38  as Assembler language for the platform-dependent part
39  - simplicity. The entire library consists of three files: a kernel header file, a file with
40  kernel source codes, and an assembler file with platform-dependent code
41  - documentation. The source codes of the library are provided with comprehensive comments,
42  as well as a guide
43  - examples. There are usage examples for all parts of the library
44 
45  History
46  v0.1 created library core as alternative to RTOS and state-machine algorithms
47  v0.2 refactoring fixed containers to unified interface
48  */

643  Item deque() { waitItem(); return mBuffer[headNext()]; }
